# RequestContext

Represents the full contextual information for an incoming API request processed by the gateway. This type aggregates all request metadata—identity, routing state, timing, headers, and payload—into a single object that flows through middleware, route matching, and backend invocation pipelines.

## API

### RequestId
`public string RequestId`

A unique identifier assigned to the request upon ingestion. This value is generated by the gateway and remains stable for the lifetime of the request, suitable for correlation across logs, traces, and downstream calls.

### Path
`public string Path`

The raw request path as received from the client, prior to any route-pattern normalization or prefix stripping. Includes the leading slash and retains original casing.

### Method
`public string Method`

The HTTP method of the request (e.g., `GET`, `POST`, `PUT`). Stored in uppercase as specified by the originating protocol adapter.

### Headers
`public Dictionary<string, string> Headers`

A case-insensitive dictionary of all request headers. Header names are normalized to lowercase. Multi-valued headers are joined with a comma separator. This collection is read/write; middleware may inject or remove headers before forwarding.

### QueryParameters
`public Dictionary<string, string> QueryParameters`

A case-sensitive dictionary of parsed query-string parameters. Values are URL-decoded once by the gateway. If the same key appears multiple times, only the first occurrence is retained.

### Body
`public string? Body`

The request body consumed and stored as a string, or `null` if the request carried no body or the body has not been read. The gateway buffers the body exactly once; subsequent reads must use this property.

### ClientIp
`public string ClientIp`

The IP address of the immediate connecting client, extracted from the transport layer. When the gateway sits behind a trusted proxy, this reflects the value from the `X-Forwarded-For` header after the gateway’s proxy resolution step.

### AuthToken
`public string? AuthToken`

The raw authentication token extracted from the request, or `null` if no token was present. The extraction strategy is configurable (e.g., `Authorization` header, custom header, or cookie). This value is set during the authentication phase and may be cleared by downstream middleware.

### ReceivedAt
`public DateTime ReceivedAt`

The UTC timestamp at which the gateway first accepted the request. Recorded before any middleware executes and expressed in `DateTime.UtcNow` precision.

### CustomData
`public Dictionary<string, object> CustomData`

A general-purpose extensibility bag for arbitrary objects. Middleware components, route handlers, and policies use this dictionary to attach computed state, caching metadata, or feature flags that travel with the request context. Keys are case-sensitive strings. The dictionary is not thread-safe for concurrent writes.

### MatchedRoute
`public GatewayRoute? MatchedRoute`

The route definition that matched the request during route resolution, or `null` if no route matched. A `GatewayRoute` carries the pattern, accepted methods, target configuration, and any route-level policies.

### SelectedTarget
`public RouteTarget? SelectedTarget`

The specific backend target selected for this request after applying load-balancing, canary rules, or failover logic. `null` when no route matched or target selection has not yet occurred.

### ClientIdentity
`public ClientIdentity? ClientIdentity`

The resolved client identity after authentication and claims mapping, or `null` for anonymous requests. A `ClientIdentity` exposes claims, roles, and the identity provider that validated the token.

### GetClientIdentifier
`public string GetClientIdentifier`

Returns the best available string identifier for the client. Priority order: the subject claim from `ClientIdentity`, then the `ClientIp`, then the string `"anonymous"`. This property never returns `null`.

### HasAuthToken
`public bool HasAuthToken`

Indicates whether a non-null, non-empty authentication token is present in `AuthToken`. Evaluated on each access; does not cache the result.

### ExtractBearerToken
`public string ExtractBearerToken`

Attempts to extract a Bearer token from the `Authorization` header. If the header value starts with `Bearer ` (case-insensitive), the substring after the scheme is returned. Throws `InvalidOperationException` when the `Authorization` header is absent or does not use the Bearer scheme.

### ElapsedTime
`public TimeSpan ElapsedTime`

The wall-clock duration elapsed since `ReceivedAt`. Computed on each access as `DateTime.UtcNow - ReceivedAt`.

## Usage

### Example 1: Middleware logging with custom data

```csharp
public async Task Invoke(RequestContext context, Func<Task> next)
{
    context.CustomData["middleware:entry"] = DateTime.UtcNow;

    await next();

    var entry = (DateTime)context.CustomData["middleware:entry"];
    var duration = DateTime.UtcNow - entry;

    Logger.Info(
        "Request {Id} {Method} {Path} completed in {Ms}ms",
        context.RequestId,
        context.Method,
        context.Path,
        duration.TotalMilliseconds);
}
```

### Example 2: Authenticated routing decision

```csharp
public RouteTarget? SelectTarget(RequestContext context)
{
    if (!context.HasAuthToken)
        return _anonymousTarget;

    var bearer = context.ExtractBearerToken();
    var identity = _tokenValidator.Validate(bearer);
    context.ClientIdentity = identity;

    if (identity.HasClaim("tenant", "premium"))
    {
        context.CustomData["routing:tier"] = "premium";
        return _premiumTarget;
    }

    return _standardTarget;
}
```

## Notes

- **Body consumption**: The gateway reads the request body stream exactly once and stores it in `Body`. Any attempt to re-read the original stream will yield an empty payload. Middleware that needs the raw bytes should access `Body` or coordinate through `CustomData`.
- **Header casing**: All header dictionary keys are lowercased during ingestion. Lookups must use lowercase keys. The original casing is not preserved.
- **Query parameter duplication**: When a query string contains duplicate keys (`?key=a&key=b`), only the first value (`a`) is stored. If full multi-value semantics are required, the raw query string must be parsed separately.
- **Thread safety**: `RequestContext` is designed to be accessed sequentially within the request pipeline. The `CustomData` dictionary and other mutable collections are not safe for concurrent writes from multiple threads. Concurrent reads are safe as long as no thread is writing.
- **`ExtractBearerToken` exception**: This property throws `InvalidOperationException` when the `Authorization` header is missing or does not use the Bearer scheme. Callers must check `HasAuthToken` or inspect headers before accessing it, or wrap the call in a try-catch block.
- **`ElapsedTime` monotonicity**: `ElapsedTime` relies on `DateTime.UtcNow` and is not monotonic; system clock adjustments can cause it to decrease. Use it for informational logging, not for precise interval measurement.
- **`GetClientIdentifier` fallback**: When both `ClientIdentity` and `ClientIp` are unavailable (e.g., misconfigured proxy resolution), the method returns `"anonymous"`. This guarantees a non-null string for logging and rate-limiting key generation.
- **`MatchedRoute` and `SelectedTarget` lifecycle**: These properties are populated at distinct pipeline stages. `MatchedRoute` is set during route resolution; `SelectedTarget` is set later during target selection. Middleware running between these stages will observe `SelectedTarget` as `null`.
